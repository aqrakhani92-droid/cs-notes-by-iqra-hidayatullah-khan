    Q2. Give short answers to the following questions.

    i. Show components of data communication with the help of a diagram.
    Data communication consists of five basic components: 1. Message (the actual data), 2. Sender (the device that sends the data), 3. Receiver (the device that receives the data), 4. Transmission Medium (the physical path like a cable or air), and 5. Protocol (the set of rules for communication). Without any of these components, successful communication cannot occur. Each component plays a vital role in ensuring that the data moves accurately from point A to point B.

    ii. Differentiate between synchronous and asynchronous transmission.
    Asynchronous transmission sends data in a steady stream of characters, where each character is preceded by a "start bit" and followed by a "stop bit" to identify the data. It is generally used for low-speed communication. Synchronous transmission, however, sends data in large blocks or frames at regular intervals timed by a clock signal. It does not use start/stop bits, making it much faster and more efficient for high-speed data transfer between computers. Asynchronous is simple and cheap, while synchronous is complex and fast.

    iii. Show all the modes of Data Communication.
    There are three modes of data communication: 1. Simplex: Data travels in only one direction (e.g., Television broadcast). 2. Half-Duplex: Data can travel in both directions but not at the same time (e.g., Walkie-talkies). 3. Full-Duplex: Data can travel in both directions simultaneously (e.g., Mobile phones). These modes define the capacity and direction of data flow between two connected devices on a network.

    iv. Differentiate between guided and unguided media.
    Guided media, also known as wired media, uses physical cables to transmit data signals. Examples include Twisted Pair, Coaxial, and Fiber Optic cables. The signal is "guided" along a specific physical path. Unguided media, or wireless media, transmits electromagnetic waves through the air or vacuum without a physical conductor. Examples include Radio waves, Microwaves, and Infrared. Guided media is more secure and has less interference, while unguided media offers mobility and covers vast geographical distances easily.

    v. Differentiate between LAN and WAN.
    A LAN (Local Area Network) is designed for a small geographical area like an office or a single building. It offers high data transfer speeds and is usually owned by a single organization. In contrast, a WAN (Wide Area Network) covers a large geographical area such as a country or even the entire world (like the Internet). WANs are slower than LANs due to the long distances and involve multiple service providers. LANs are used for local resource sharing, while WANs are used for global communication.

    vi. What is OSI Model?
    The OSI (Open Systems Interconnection) model is a conceptual framework developed by ISO to standardize the functions of a telecommunication or networking system. It divides the complex process of networking into seven logical layers: Physical, Data Link, Network, Transport, Session, Presentation, and Application. Each layer performs a specific task and communicates with the layers above and below it. The main purpose of the OSI model is to ensure that different networking products from different vendors can work together seamlessly.

    vii. What is TCP/IP Model?
    The TCP/IP (Transmission Control Protocol/Internet Protocol) model is the standard protocol suite used for communication across the internet and most modern networks. Unlike the OSI model's 7 layers, TCP/IP typically uses 4 or 5 layers: Network Interface, Internet, Transport, and Application. It is a practical model designed to solve real-world networking problems. TCP handles the reliable delivery of data by breaking it into packets and reassembling them, while IP handles the addressing and routing of those packets to their destination.

    viii. Compare TCP/IP with OSI Model.
    The OSI model is a theoretical model used for education and troubleshooting, while TCP/IP is a practical model used to run the internet. OSI has seven distinct layers, whereas TCP/IP combines some of these into four or five layers. For example, the Session, Presentation, and Application layers of OSI are combined into a single Application layer in TCP/IP. TCP/IP is more widely used because it was developed specifically for the Internet, whereas OSI was developed to be a general-purpose networking standard.

    ix. Differentiate between circuit switched and packet switched networks.
    Circuit switching establishes a dedicated physical connection between the sender and receiver before data is sent (e.g., traditional phone calls). Once established, no one else can use that path. Packet switching does not require a dedicated path; instead, data is broken into small "packets" that can take different routes to the destination and are reassembled there (e.g., Internet data). Packet switching is more efficient because it uses the network's capacity more flexibly, while circuit switching guarantees constant bandwidth.

    x. Briefly describe IP Addressing.
    An IP (Internet Protocol) address is a unique numerical label assigned to every device connected to a computer network. It serves two main functions: identifying the host or network interface and providing the location of the device in the network. There are two versions: IPv4 (e.g., 192.168.1.1) which uses 32 bits, and IPv6 (e.g., 2001:0db8:...) which uses 128 bits. IP addresses allow routers to direct data packets to the correct destination across the global internet, similar to how a home address works for mail.

    Q3. Give detailed answers to the following questions.

    i. Explain various modes of data communication.

    Data communication mode refers to the direction of signal flow between two linked devices. There are three primary modes:

    1. Simplex Mode

    Communication is unidirectional. Only one device can transmit, and the other can only receive. Example: A keyboard can only send input; a monitor can only receive output. Broadcast radio and TV are also examples.

    2. Half-Duplex Mode

    Communication is bidirectional, but not at the same time. While one device is sending, the other must wait. Example: A Walkie-talkie. It is efficient for situations where simultaneous communication is not required.

    3. Full-Duplex Mode

    Communication is bidirectional and simultaneous. Both stations can transmit and receive data at the same time. Example: A telephone conversation. This is the most efficient mode for modern high-speed networks.

    ii. What is guided media? Explain different types.

    Guided media provides a physical path for signals to follow. The signal is contained within the boundaries of the medium.

    1. Twisted Pair Cable

    Consists of pairs of copper wires twisted together to reduce electromagnetic interference. It is cheap and used for LANs (CAT5, CAT6).

    2. Coaxial Cable

    Consists of a central copper core surrounded by an insulator, a metal shield, and an outer jacket. It carries high-frequency signals and is used for Cable TV.

    3. Fiber Optic Cable

    Uses glass or plastic threads to transmit data as pulses of light. It is extremely fast, supports huge bandwidth, and is immune to electrical interference. It is the backbone of modern internet.

    iv. Write notes on switch, router and gateway.

    1. Switch

    A switch is an intelligent device used to connect multiple devices within a single LAN. Unlike a hub, a switch sends data only to the specific port of the destination device using its MAC address, which reduces network traffic and improves speed.

    2. Router

    A router is a device that connects different networks together (e.g., connecting your home LAN to the Internet). It uses IP addresses to forward data packets between networks using the most efficient path.

    3. Gateway

    A gateway is a network node that connects two networks that use different protocols. It acts as a "translator" between different network architectures, allowing them to communicate seamlessly.

    vii. Describe the seven layers of OSI Model.

    The OSI model organizes network communication into seven layers:

    1. Physical Layer: Deals with physical cables, bits, and electrical signals.
    2. Data Link Layer: Responsible for error-free transfer of frames and MAC addressing.
    3. Network Layer: Handles routing of packets and IP addressing.
    4. Transport Layer: Ensures reliable end-to-end delivery of data (TCP/UDP).
    5. Session Layer: Manages and terminates sessions between applications.
    6. Presentation Layer: Handles data encryption, compression, and formatting.
    7. Application Layer: Provides network services directly to the user (HTTP, FTP).

    viii. What is TCP/IP? Explain TCP/IP Protocol Suite (Architecture, Applications & Ports).

    The TCP/IP suite is the most popular network protocol used for internet communication. It is designed to be highly reliable and scalable.

    1. TCP/IP Architecture Layers

    • Application Layer: Provides high-level protocols for user interaction.
    • Transport Layer: Provides flow control and error correction (TCP/UDP).
    • Internet Layer: Handles logical addressing and routing (IP).
    • Network Interface Layer: Manages physical transmission of data.

    2. Common Applications and Ports

    Each application in the TCP/IP suite is assigned a unique "Port Number" so the computer knows which software should receive the incoming data:

    Application / Protocol Standard Port Description
    HTTP 80 Used for transferring web pages.
    HTTPS 443 Secure web browsing (encrypted).
    FTP 20, 21 Used for transferring files between computers.
    SMTP 25 Used for sending emails.
    DNS 53 Translates domain names (google.com) to IP addresses.
