Q2. Give short answers to the following questions.

i. What is the function of ALU in the computer?
The Arithmetic and Logic Unit (ALU) is the core component of the CPU where actual data processing takes place. Its primary function is to perform all mathematical calculations such as addition, subtraction, multiplication, and division. Additionally, it handles logical operations, which involve comparing two data items to determine if one is greater than, less than, or equal to the other. Every instruction executed by the computer that involves calculation or decision-making is processed through the ALU.

ii. What is the function of control unit in the computer?
The Control Unit (CU) acts as the supervisor or manager of the entire computer system. It does not process data itself but coordinates the flow of information between the CPU, memory, and I/O devices. It fetches instructions from the main memory, decodes them to understand the required action, and then issues control signals to other components (like the ALU or Registers) to execute the command. It ensures that operations are performed in the correct order and at the right time.

iii. What is a microprocessor?
A microprocessor is a single integrated circuit (IC) or silicon chip that contains all the functions of a Central Processing Unit (CPU). It is the "brain" of modern computers, consisting of millions of transistors that handle data processing and control tasks. It is responsible for executing program instructions and managing the hardware components. Microprocessors are used in a wide range of devices, including PCs, laptops, smartphones, and embedded systems, and their power is usually measured by clock speed and core count.

iv. Define system bus.
A system bus is a high-speed communication pathway on the motherboard that connects the CPU to other major components like RAM and input/output controllers. It consists of a set of parallel wires or tracks that transmit electrical signals representing data, memory addresses, and control commands. The system bus is typically divided into three specialized buses: the Data Bus for moving data, the Address Bus for identifying memory locations, and the Control Bus for transmitting timing and coordination signals between components.

v. Write short note on Memory Buffer Register (MBR).
The Memory Buffer Register (MBR), also known as the Memory Data Register (MDR), is a specialized high-speed storage location within the CPU. Its role is to temporarily hold the actual data or instruction that is either being fetched from the main memory (RAM) or is being sent from the CPU to be stored in the memory. It acts as a buffer that allows the processor to work efficiently while the data transfer between the fast CPU and the relatively slower main memory is being completed.

vi. What is CPU operation?
CPU operation refers to the fundamental process by which the processor executes a single instruction, often called the Machine Cycle. This operation consists of four repetitive stages: Fetch (getting the instruction from memory), Decode (translating the instruction into signals), Execute (performing the actual task in the ALU or other units), and Write-back (storing the result). The CPU performs these operations billions of times per second, allowing it to run software, process user input, and manage system resources simultaneously.

vii. What is meant by an instruction?
In computer science, an instruction is a basic command or order given to the processor to perform a specific task. These tasks can include moving data, performing a calculation, or jumping to a different part of a program. Instructions are written in a language the hardware understands (machine code). A complete set of these instructions forms a computer program. Each instruction usually contains an operation code (Op-code) and one or more memory addresses or data values known as operands.

viii. Differentiate between op-code and operand?
An Op-code (Operation Code) is the part of a machine language instruction that specifies the operation to be performed, such as ADD, SUB, or MOV. It essentially tells the CPU "what to do." An Operand is the data or the memory address upon which the operation is performed. It tells the CPU "what to do it with." For example, in the command `ADD 5, 10`, `ADD` is the Op-code, while `5` and `10` are the operands. Every instruction must have an Op-code, but the number of operands can vary.

ix. Show computer instruction format with the help of a diagram and label its parts.
The standard format for a computer instruction is divided into two primary fields: 1. Op-code Field: This field contains the code for the operation to be performed (like binary for ADD). 2. Address/Operand Field: This field contains the data or the memory location of the data to be used in the operation. (In a 16-bit instruction, for instance, the first 4 bits might be the Op-code and the remaining 12 bits represent the Operand address).

x. Differentiate between CISC and RISC processor architecture.
CISC (Complex Instruction Set Computing) is an architecture where the processor can execute complex instructions that may take multiple clock cycles (e.g., Intel x86). It focuses on hardware complexity to reduce program code size. RISC (Reduced Instruction Set Computing) uses a small, highly optimized set of simple instructions that mostly execute in a single clock cycle (e.g., ARM). RISC focuses on software/compiler efficiency and is generally faster and more power-efficient, making it ideal for mobile devices.

Q3. Give detailed answers to the following questions.

i. What is CPU? Describe basic components of CPU in detail.

The Central Processing Unit (CPU) is often referred to as the "brain" of the computer. It is responsible for interpreting and executing instructions from the hardware and software.

1. Arithmetic and Logic Unit (ALU)

The ALU is the calculating part of the CPU. It handles all mathematical operations (Addition, Subtraction) and logical operations (Comparing values). It is the actual "processor" inside the CPU.

2. Control Unit (CU)

The Control Unit does not process data; instead, it directs the flow of data. It fetches instructions from memory, decodes them, and manages the execution by sending signals to other parts of the system.

3. Registers

Registers are tiny, high-speed storage locations within the CPU chip. They hold data, instructions, and addresses that are needed immediately by the processor. They are the fastest memory in the entire system.

4. Internal CPU Bus

This is the communication path that connects the ALU, CU, and registers together, allowing them to exchange data at extremely high speeds during the instruction cycle.

ii. What is a register? Explain different types of registers.

Registers are very small, extremely fast memory cells located directly inside the CPU. They are used to store data and instructions temporarily for immediate processing.

1. General Purpose Registers

These are used for general data movement and arithmetic. Common examples include AX (Accumulator), BX (Base), CX (Counter), and DX (Data) registers.

2. Special Purpose Registers

  • Program Counter (PC): Holds the address of the next instruction to be fetched from memory.
  • Memory Address Register (MAR): Holds the address of the memory location that the CPU is currently accessing.
  • Memory Buffer Register (MBR): Holds the actual data being read from or written to memory.
  • Instruction Register (IR): Holds the current instruction while it is being decoded and executed.

iii. What is a system bus? Explain different types of buses used in computers.

A system bus is a set of parallel electrical paths (wires) on the motherboard that allow the CPU to communicate with other parts of the computer.

1. Data Bus

The Data Bus carries the actual data (bits) between the CPU, memory, and input/output devices. It is bi-directional, meaning data can move in both directions.

2. Address Bus

The Address Bus carries the memory address of the location where data is stored or needs to be sent. It is uni-directional, as the CPU always generates the addresses.

3. Control Bus

The Control Bus carries control and timing signals to coordinate all activities. Signals like "Read," "Write," and "Interrupt" travel through this bus to keep the system synchronized.

iv. Explain different types of CPU instructions. Also give examples.

CPU instructions are commands that tell the processor what to do. They are broadly classified into four categories:

1. Data Transfer Instructions

These instructions move data between registers or between memory and registers. Example: `MOV` (Move), `PUSH`, `POP`.

2. Arithmetic and Logic Instructions

These perform math or logical comparisons. Example: `ADD`, `SUB` (Subtract), `AND`, `OR`, `NOT`.

3. Program Control Instructions

These change the flow of a program, allowing it to skip sections or repeat them. Example: `JMP` (Jump), `LOOP`, `CALL`.

4. Input/Output (I/O) Instructions

These instructions handle data exchange with external devices like keyboards or printers. Example: `IN`, `OUT`.

v. What is Instruction Cycle? Explain different phases of CPU instruction cycle.

The Instruction Cycle (also known as the Machine Cycle) is the continuous process by which the CPU executes a single instruction from a program.

1. Fetch Phase

The CPU retrieves the instruction from the RAM using the address stored in the Program Counter. The instruction is then placed in the Instruction Register (IR).

2. Decode Phase

The Control Unit breaks down the instruction into binary signals to understand which operation (like ADD or MOVE) needs to be performed.

3. Execute Phase

The CPU performs the action. If it's a calculation, the ALU carries it out. If it's a jump, the PC is updated with a new address.

4. Write-back Phase

The result of the execution is stored back into a register or main memory so it can be used later by other instructions.
