Q2. Give short answers to the following questions.

i. Define the terms database and DBMS.
A database is an organized collection of related data stored in a way that it can be easily accessed, managed, and updated. It represents real-world entities like students or employees. A Database Management System (DBMS) is the software used to create and manage these databases. It provides an interface for users to perform tasks like adding, deleting, or retrieving data efficiently.

ii. Give three examples of data and information.
Data is raw facts, while information is processed data. 1) Raw marks of a student (Data) converted into a Grade (Information). 2) Daily sales figures (Data) compiled into a Monthly Profit Report (Information). 3) A list of temperatures (Data) used to calculate the Average Weather for a week (Information). Thus, data serves as input and information is the useful output.

iii. Write three disadvantages of file management system.
1) Data Redundancy: The same data is often duplicated in different files, wasting storage space. 2) Data Inconsistency: Updating data in one file might not update it in another, leading to conflicting records. 3) Lack of Security: It is difficult to restrict access to specific parts of the data, making it vulnerable to unauthorized changes or data theft.

iv. Give any four advantages of database system.
1) Data Integration: All data is stored in one central location for easy access. 2) Reduced Redundancy: DBMS ensures that data is not repeated unnecessarily. 3) Data Integrity: It enforces rules to ensure data accuracy and consistency. 4) Better Security: Access can be restricted to authorized users through passwords and permission levels.

v. Differentiate between DDL and DML.
Data Definition Language (DDL) is used to define the structure or schema of the database, such as creating or deleting tables. Examples include CREATE and DROP. Data Manipulation Language (DML) is used to manage the actual data within those structures. Examples include SELECT, INSERT, and UPDATE. While DDL deals with the "container," DML deals with the "content" inside it.

vi. What is data model?
A data model is a collection of conceptual tools or rules used to describe how data is represented, organized, and stored in a database. it defines the relationships between data elements and sets constraints on how data can be manipulated. Common examples include the Relational Model, Hierarchical Model, and Network Model, each providing a different logical view of the database.

vii. Differentiate between cardinality and modality.
Cardinality refers to the maximum number of times an instance in one entity can be associated with instances in another entity (e.g., One-to-One, One-to-Many). Modality (or participation) refers to the minimum number of times an instance must participate in a relationship, often indicated as being either Optional (0) or Mandatory (1).

viii. What is primary key and foreign key?
A Primary Key is a unique field that identifies each record in a table and cannot contain null values. For example, "Student_ID." A Foreign Key is a field in one table that refers to the Primary Key in another table. It is used to establish a link or relationship between the two tables, ensuring referential integrity across the database system.

ix. What is Relation? What is the role of DBA?
In database terms, a Relation is another name for a Table, consisting of rows (tuples) and columns (attributes). The Database Administrator (DBA) is a person responsible for the overall management of the database. Their role includes installing software, ensuring data security, managing user access, performing backups, and optimizing the database performance for the organization.

ix. What is the role of DBA?
The Database Administrator (DBA) is a technical professional responsible for the overall management and smooth functioning of a database system. Their key roles include:

  • Installation: Setting up the DBMS software and hardware.
  • Security: Protecting data from unauthorized access by managing user accounts and passwords.
  • Backup & Recovery: Creating regular backups and restoring data if the system fails.
  • Performance Tuning: Optimizing the database speed so users can retrieve data quickly.
  • Integrity: Ensuring that the data entered is accurate and follows the database rules.

Q3. Give detailed answers to the following questions.

i. Explain different types of database models.
Database models define the logical structure of a database. The three main types are:
1. Hierarchical Model: Data is organized into a tree-like structure. Each child record has only one parent, but a parent can have multiple children. It is simple but rigid.
2. Network Model: Similar to the hierarchical model, but a child record can have multiple parents. This forms a graph-like structure and is more flexible for complex relationships.
3. Relational Model: The most common model where data is stored in tables (relations). Tables are linked through common fields. It is highly flexible and easy to use with SQL.
4. Object-Oriented Model: Represents information in the form of objects as used in object-oriented programming. It is ideal for complex data types like audio and video.

ii. What is SQL? Explain its types.
Structured Query Language (SQL) is the standard language for interacting with relational databases. It allows users to create, search, update, and delete data. It is divided into three main categories:
1. DDL (Data Definition Language): Used to define the database structure. Commands include CREATE (to make tables), ALTER (to modify), and DROP (to delete).
2. DML (Data Manipulation Language): Used to handle the data itself. Commands include INSERT (to add rows), SELECT (to retrieve data), UPDATE (to change data), and DELETE (to remove rows).
3. DCL (Data Control Language): Used to control access and permissions. Commands include GRANT (to give access) and REVOKE (to take access away). SQL provides a unified way to manage complex data systems.

iii. Explain various steps of planning a database.
Planning a database is a systematic process to ensure data accuracy. The steps include:
1. Requirement Analysis: Identifying what data needs to be stored and how users will interact with it.
2. Conceptual Design: Creating a high-level plan, often using Entity-Relationship Diagrams (ERD), to show entities and their links.
3. Logical Design: Converting the ERD into a set of tables and applying rules to ensure data is organized efficiently.
4. Physical Design: Deciding how the data will be stored on physical disks and choosing the right DBMS software.
5. Implementation: Writing the SQL code to create the tables and loading the initial data. Proper planning prevents future errors and ensures scalability.

iv. What is a relation? Explain the degree of relationships with examples.
A relation is a two-dimensional table used to store data in a relational database. Relationships define how these tables connect to each other. The degree refers to the number of entities involved:
1. Unary Relationship: A relationship between instances of a single entity (e.g., an Employee manages another Employee).
2. Binary Relationship: The most common type, involving two entities (e.g., a Student enrolls in a Course).
3. Ternary Relationship: Involves three entities simultaneously (e.g., a Doctor prescribes a Medicine to a Patient).
Understanding these degrees is crucial for building accurate Entity-Relationship Diagrams and database structures.

v. What is ERD? Draw an ERD for the following systems.
An Entity-Relationship Diagram (ERD) is a visual representation of the entities within a system and the relationships between them. It uses symbols like Rectangles (Entities), Ovals (Attributes), and Diamonds (Relationships).
(a) Hospital Management System: Entities include Patient, Doctor, and Ward. A Doctor "treats" a Patient; a Patient "occupies" a Ward.
(b) Airline Booking System: Entities include Passenger, Flight, and Ticket. A Passenger "books" a Ticket; a Flight "carries" Passengers. These diagrams help developers understand the logic of the database before writing code.



(a) Hospital Management System:
In this system, the main entities are Doctor, Patient, and Ward.
  • A Doctor treats a Patient (One-to-Many: One doctor can treat many patients).
  • A Patient is admitted to a Ward (Many-to-One: Many patients can stay in one ward).
  • Attributes for Doctor include Doctor_ID, Name, Specialization.
  • Attributes for Patient include Patient_ID, Name, Disease.


(b) Airline Booking System:
In this system, the main entities are Passenger, Flight, and Ticket.
  • A Passenger buys a Ticket (One-to-Many: One passenger can have multiple tickets for different journeys).
  • A Ticket is issued for a specific Flight (Many-to-One: Many tickets belong to one flight).
  • Attributes for Passenger include Passport_No, Name, Contact.
  • Attributes for Flight include Flight_No, Destination, Departure_Time.

vi. Describe different steps of Transformation of E-R Model into Relational Schema.
Transforming an ERD into a Relational Schema involves several steps:
1. Represent Entities as Tables: Each entity becomes a separate table.
2. Represent Attributes: Each attribute of the entity becomes a column in the table.
3. Identify Primary Keys: Choose a unique identifier for each table.
4. Handle Relationships: For One-to-Many, add the primary key of the "One" side as a foreign key in the "Many" side. For Many-to-Many, create a new "Link" table containing the primary keys of both participating entities. This ensures that the logical diagram is correctly implemented into actual database tables.

vii. What is Normalization? Explain 1NF, 2NF, 3NF.
Normalization is the process of organizing data in a database to reduce redundancy and improve data integrity. It involves dividing large tables into smaller ones.
1. First Normal Form (1NF): A table is in 1NF if every cell contains only a single (atomic) value and there are no repeating groups.
2. Second Normal Form (2NF): The table must be in 1NF, and all non-key attributes must be fully dependent on the primary key (no partial dependency).
3. Third Normal Form (3NF): The table must be in 2NF, and there should be no transitive dependency, meaning non-key attributes should not depend on other non-key attributes. Normalization ensures the database is efficient and free of update anomalies.
