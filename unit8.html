Q2. Give short answers to the following questions.

i. Name different types of database management systems.
There are several types of Database Management Systems (DBMS) based on how they organize data. The most common ones include Relational DBMS (RDBMS) like MS Access and MySQL, Hierarchical DBMS, Network DBMS, and Object-Oriented DBMS. Each type has a unique way of storing and retrieving data depending on the complexity and requirements of the organization.

ii. What is OpenOffice Base?
OpenOffice Base is a fully featured desktop database management system designed to meet the needs of a broad array of users. It is an open-source application, meaning it is free to use and modify. It allows users to create tables, queries, forms, and reports, and can act as a front-end for various database engines like MySQL or Adabas D.

iii. How forms are useful in database?
Forms provide a user-friendly interface for interacting with the database. They are useful because they allow users to enter, edit, and view data without needing to look at the complex table structures. Forms can include buttons, text boxes, and labels that guide the user, making data entry more accurate and reducing the chance of technical errors.

iv. Name common Access database objects.
Microsoft Access relies on several core objects to function. The four most common objects are: 1) Tables (to store data), 2) Queries (to search and retrieve specific data), 3) Forms (to provide a user interface for data entry), and 4) Reports (to format and summarize data for printing or presentation).

v. How primary key is assigned in a table?
In MS Access, a primary key can be assigned by selecting a field in "Design View" and clicking the "Primary Key" button on the Ribbon. This ensures that every record in that table is unique. Alternatively, Access often prompts the user to create a primary key automatically if they save a table without one, usually creating an "ID" field with the Autonumber data type.

vi. What is a report?
A report is a database object used to present information from tables or queries in a structured and professional format. Unlike forms or tables, reports are specifically designed for output, such as printing or converting to PDF. They can group data, calculate totals or averages, and use visual elements like headers and footers to make information easy to read.

vii. How relationship can be edited in Access?
Relationships can be edited by going to the "Database Tools" tab and clicking on "Relationships." This opens a window showing all connected tables. By double-clicking the line connecting two tables, the "Edit Relationships" dialog box appears. Here, users can change the relationship type or enable features like "Enforce Referential Integrity" and "Cascade Update/Delete."

viii. Name different navigation buttons in a form.
Navigation buttons are located at the bottom of a form window and allow users to move through records. The common buttons include: 1) First Record, 2) Previous Record, 3) Current Record (showing the record number), 4) Next Record, 5) Last Record, and 6) New (Blank) Record button used for adding a new entry to the database.

Q3. Give detailed answers to the following questions.

i. Explain the following database objects: (a) Tables (b) Forms (c) Queries (d) Reports.
Tables: These are the foundation of any database. They store data in rows (records) and columns (fields). Each table represents a specific subject, such as "Students" or "Orders."
Forms: Forms act as the interface between the user and the data. They allow for easy data entry and viewing, often including navigation controls and data validation features.
Queries: Queries are used to ask questions of the database. They can filter records based on specific criteria, join data from multiple tables, and even perform calculations.
Reports: Reports are designed for output. They organize and summarize data in a professional layout suitable for printing, often including headers, footers, and calculated fields like "Total Sales."

ii. Explain different data types used in Access.
MS Access provides various data types to ensure data accuracy:
1. Short Text: Stores up to 255 alphanumeric characters (names, addresses).
2. Long Text: Used for large amounts of text (sentences or paragraphs).
3. Number: Stores numeric data for mathematical calculations.
4. Date/Time: Specifically for calendar dates and clock times.
5. Currency: Used for monetary values to prevent rounding errors.
6. AutoNumber: Automatically generates a unique number for every new record.
7. Yes/No: A logical data type for "True/False" or "On/Off" values.

iii. Describe two methods of creating a table in Access.
1. Datasheet View: This is the simplest method where you enter data directly into a spreadsheet-like grid. Access automatically assigns data types based on the values you type. It is best for quick and simple tables.
2. Design View: This method gives the user complete control. You manually define field names, choose specific data types, and set field properties (like field size or validation rules). This is the preferred method for professional database development as it ensures the structure is optimized before data is added.

iv. Explain the process of creating forms in Access.
Forms can be created using several tools. The Form Tool creates a basic form automatically from a selected table. The Form Wizard provides a step-by-step process where you select specific fields, choose a layout (like Columnar or Tabular), and pick a style. For advanced users, Form Design allows you to build a form from scratch, placing text boxes, buttons, and images exactly where you want them. Once created, forms can be used in "Form View" for data entry or "Layout View" for visual adjustments.

v. Write the steps for creating relationships between tables.
To create a relationship: 1) Open the "Database Tools" tab and click "Relationships." 2) Add the tables you want to link using the "Show Table" dialog. 3) Drag the primary key field from one table and drop it onto the matching foreign key field in the other table. 4) In the "Edit Relationships" window, check "Enforce Referential Integrity" if needed. 5) Click "Create." A line will appear between the tables indicating that the link is established.

vi. Explain different types of queries used in Access.
1. Select Query: The most common type, used to retrieve specific data from one or more tables.
2. Action Query: Used to change or move data (e.g., Update Query, Delete Query, or Append Query).
3. Parameter Query: A query that asks the user for input (like a date range) before it runs.
4. Crosstab Query: Used to calculate and restructure data for easier analysis, similar to a pivot table in Excel.

vii. How can One-to-Many relationship be created between two tables? Give example.
A One-to-Many relationship exists when a record in Table A can match many records in Table B, but a record in Table B matches only one in Table A. It is created by placing the Primary Key of the "One" side into the "Many" side as a Foreign Key. Example: A "Customer" table and an "Orders" table. One customer can place many orders, but each order belongs to only one specific customer. The "CustomerID" in the Orders table acts as the link.

viii. What is a Report? How report is created using wizard option in Access 2016?
A report is a formatted summary of data for printing. To use the Report Wizard: 1) Go to the "Create" tab and click "Report Wizard." 2) Select the table or query and the specific fields you want to include. 3) Choose grouping levels (e.g., group by city). 4) Select a sorting order (e.g., by name). 5) Pick a layout (Stepped, Block, or Outline). 6) Give the report a title and click "Finish." Access will then generate the report for you to review.
